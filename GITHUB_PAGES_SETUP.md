# GitHub Pages — Автоматический деплой сайта

## Что это даёт
После каждого `git push` сайт автоматически обновляется на `www.sorbgaz.ru`

---

## ШАГ 1: Создать CNAME файл ✅ (уже сделано)
Файл `CNAME` создан в корне репозитория с содержимым:
```
www.sorbgaz.ru
```

---

## ШАГ 2: Настроить GitHub Pages (в браузере)

### 2.1. Открыть репозиторий
Перейти на: https://github.com/te213674/uog-site

### 2.2. Включить GitHub Pages
1. Кликнуть **Settings** (вкладка над файлами)
2. В меню слева найти **Pages**
3. В секции **Source**:
   - Выбрать **Deploy from a branch**
   - Branch: **main** / папка: **/ (root)**
   - Нажать **Save**

### 2.3. Привязать свой домен
1. В секции **Custom domain** ввести: `www.sorbgaz.ru`
2. ✅ Поставить галочку **Enforce HTTPS** (SSL включится автоматически через Let's Encrypt)
3. Сохранить

---

## ШАГ 3: Настроить DNS (у регистратора домена)

### Где находится панель управления доменом?
Узнайте, где вы покупали домен (Timeweb, reg.ru, Cloudflare и т.д.)

### Создать DNS записи:

#### Вариант A — CNAME для www:
| Тип | Имя | Значение |
|-----|-----|----------|
| CNAME | www | te213674.github.io |

#### Вариант B — Если нужен редирект sorbgaz.ru → www.sorbgaz.ru:
| Тип | Имя | Значение |
|-----|-----|----------|
| A | @ | 185.199.108.153 |
| A | @ | 185.199.109.153 |
| A | @ | 185.199.110.153 |
| A | @ | 185.199.111.153 |
| CNAME | www | te213674.github.io |

---

## ШАГ 4: Подтвердить владение доменом (если потребуется)

GitHub может попросить добавить TXT запись. Если увидите такое требование:
1. Добавить TXT запись с указанным значением
2. Подождать 5-30 минут
3. Нажать **Save** / **Verify**

---

## ШАГ 5: Запушить изменения

После выполнения всех шагов:

```bash
git add CNAME
git commit -m "Add CNAME for custom domain"
git push
```

---

## Проверка

1. Через 1-5 минут после пуша сайт появится на: https://www.sorbgaz.ru
2. В браузере должен быть зелёный замочек 🔒

---

## Если что-то не работает

### HTTPS не работает:
- Подождать до 24 часов (первый SSL может обновляться долго)
- Проверить что CNAME указывает правильно

### Сайт не открывается:
- Проверить DNS записи
- Убедиться что GitHub Pages включён (Settings → Pages)

### Ошибка 404:
- Проверить что в репозитории есть index.html в корне
